# 视频软件开发培训方案

> 面向本地/流媒体播放器（如 Isshin Player）的必备知识体系。

---

## 一、培训目标与阶段

| 阶段 | 目标 | 时长建议 |
|------|------|----------|
| 基础 | 理解音视频基础、项目架构与开发流程 | 1–2 周 |
| 进阶 | 掌握播放链路、后端 API、前端状态与 Electron 集成 | 2–3 周 |
| 实战 | 能独立完成需求开发、排错与打包发布 | 持续 |

---

## 二、必备知识内容

### 2.1 音视频基础

- **容器与编码**：容器格式（MP4、MKV、WebM 等）与编码（H.264、AAC 等）的区别；MIME 类型与 Content-Type 的对应。
- **流式播放**：HTTP Range 请求（`Range`、`206 Partial Content`、`Content-Range`）；HLS（.m3u8）与普通 MP4 的差异及前端识别方式。

### 2.2 项目架构

- **前端**：Vite + React 18 + TypeScript + Tailwind；HashRouter；ResourcesProvider 管理目录、列表、当前文件、播放状态。
- **后端**：Node.js + Express；`/getFiles`（目录+音视频列表）、`/video`（Range 流）；MIME 分类（mimeTypes.js）。
- **Electron**：主进程启动 Express 再开窗口；开发加载 Vite，生产加载 frontend/dist；electron-builder 打包。

### 2.3 后端能力

- `/getFiles`：path、keyword；只返回目录与音视频；结构 `{ name, type, path }`。
- 流接口：根据 `Range` 返回 206，`fs.createReadStream(path, { start, end })`；正确设置 Content-Type、Accept-Ranges、Content-Range。
- 路径校验、错误码、CORS 与缓存头。

### 2.4 前端能力

- 状态：当前路径、分类、文件列表、当前文件、播放模式；与 `/getFiles`、播放器 src 的联动。
- 播放器：`<video>`/`<audio>` src 指向后端流地址；HLS 与普通视频区分；快捷键（空格、上下首、M 等）。
- 列表与搜索：右侧列表来自 getFiles；关键词请求与高亮、选中同步。

### 2.5 Electron

- 启动顺序：Express → BrowserWindow；开发/生产 URL 与 base。
- 打包：productName、extraResources（后端）；Windows nsis/portable、macOS dmg/zip。

### 2.6 调试排错

- Network 查看 /getFiles、/video 的 Range、206、Content-Type。
- 路径、CORS、MIME 错误导致的无法播放或列表异常。

---

## 三、学习路径

1. **第 1 周**：音视频基础 + 本地 `yarn dev`，观察目录、列表、播放及 Network 请求。
2. **第 2 周**：阅读 backend/index.js、mimeTypes.js；前端 getFiles、video src、ResourcesProvider。
3. **第 3 周**：做一个小需求（如新快捷键或过滤规则），并完成打包验证。

---

## 四、考核建议

- 理论：Range/206、HLS 与 MP4、MIME 与扩展名。
- 实操：在现有代码中新增一个 API 或播放行为并说明改动。
- 排错：给定「无法播放」或「列表为空」场景，能定位路径/MIME/Range/CORS。
